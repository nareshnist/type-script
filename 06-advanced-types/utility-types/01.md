# 🔥 1. `Partial<T>` — Update API case

### 🎯 Scenario:

User profile update chestunnav, kani anni fields pampakudadhu

```ts
type User = {
  id: number;
  name: string;
  email: string;
}
```

### ❌ Without Partial:

```ts
function updateUser(user: User) {}
```

👉 anni fields compulsory 😓

---

### ✅ With Partial:

```ts
function updateUser(user: Partial<User>) {
  console.log(user);
}

updateUser({ name: "Naresh" }) // ✅ only name enough
```

👉 **Use case:** PATCH API calls

---

# 🔥 2. `Pick<T, K>` — UI lo limited data

### 🎯 Scenario:

UI lo only name display cheyali

```ts
type User = {
  id: number;
  name: string;
  email: string;
}
```

### ✅

```ts
type UserName = Pick<User, "name">

const user: UserName = {
  name: "Naresh"
}
```

👉 **Use case:** table columns, dropdowns

---

# 🔥 3. `Omit<T, K>` — Sensitive data remove

### 🎯 Scenario:

Password ni UI ki pampakudadhu

```ts
type User = {
  id: number;
  name: string;
  password: string;
}
```

### ✅

```ts
type SafeUser = Omit<User, "password">

const user: SafeUser = {
  id: 1,
  name: "Naresh"
}
```

👉 **Use case:** API response sanitize

---

# 🔥 4. `Readonly<T>` — Immutable data

### 🎯 Scenario:

Config data change kakudadhu

```ts
type Config = {
  apiUrl: string;
}
```

### ✅

```ts
const config: Readonly<Config> = {
  apiUrl: "https://api.com"
}

// config.apiUrl = "new" ❌ error
```

👉 **Use case:** constants, environment config

---

# 🔥 5. `Required<T>` — Form validation

### 🎯 Scenario:

Optional fields ni mandatory cheyali

```ts
type Form = {
  name?: string;
  email?: string;
}
```

### ✅

```ts
type RequiredForm = Required<Form>

const form: RequiredForm = {
  name: "Naresh",
  email: "test@gmail.com"
}
```

👉 **Use case:** submit time validation

---

# 🔥 6. `Record<K, T>` — Dynamic object (roles example 🔥)

### 🎯 Scenario:

Roles ki users assign cheyadam

```ts
type Roles = "admin" | "user"
```

### ✅

```ts
const users: Record<Roles, string> = {
  admin: "Naresh",
  user: "Ravi"
}
```

👉 **Use case:**

* role-based systems
* status mapping
* config maps
