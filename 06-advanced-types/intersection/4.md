# 🔵 `|` (Union Type) ante enti?

👉 **Union (`|`) means:**
👉 **“Either this OR that”**

---

# 🟢 Simple Example

```ts
type A = {
  name: string;
};

type B = {
  age: number;
};

type C = A | B;
```

---

## ✅ Valid cases:

```ts
const obj1: C = { name: "Naresh" }; // ✅ A
const obj2: C = { age: 25 };        // ✅ B
```

---

## ⚠️ Also valid (important)

```ts
const obj3: C = {
  name: "Naresh",
  age: 25
}; // ✅ still valid
```

👉 Why?
Because it satisfies **at least one type**

---

# 🔴 Key Rule

👉 Union lo:
✔ **Minimum one type satisfy ayite saripothundi**

---

# 🟡 Telugu Explanation

👉 `|` ante:

👉 **“idi lekapothe adi”**

👉 Example:

* A → name
* B → age

👉 Final:
➡️ name unna saripothundi
➡️ age unna saripothundi
➡️ rendu unna kuda problem ledu

---

# 🔥 Real-world Example

## 🟢 API Response

```ts
type Success = {
  data: string;
};

type Error = {
  message: string;
};

type Response = Success | Error;
```

---

## Usage:

```ts
function handle(res: Response) {
  if ("data" in res) {
    console.log(res.data);
  } else {
    console.log(res.message);
  }
}
```

# 🟣 Compare with `&` (Very Important)

| Feature | `&` (Intersection) | `|` (Union) |
|--------|------------------|------------|
| Meaning | AND | OR |
| Required fields | All | Any one |
| Use case | Combine | Choose |

---

## Example Difference

```ts
type A = { name: string };
type B = { age: number };
```

---

### `&`

```ts
let x: A & B = {
  name: "Naresh",
  age: 25
}; // both required
```

---

### `|`

```ts
let y: A | B = {
  name: "Naresh"
}; // only one enough
```

## Correct usage:

### ✔ Use `&` when:

👉 rendu kavali (combine)

### ✔ Use `|` when:

👉 okati saripothundi (either)
