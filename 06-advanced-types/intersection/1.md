Intersection types in **TypeScript** are a powerful concept. Let’s break it down step by step—from **beginner → intermediate → expert** level—with clear understanding.

---

# 🟢 1. Beginner Level – What is Intersection Type?

👉 **Simple Definition:**
Intersection type means **combining multiple types into one**.

👉 Syntax:

```ts
type NewType = TypeA & TypeB;
```

👉 Meaning:
➡️ The new type will have **all properties from both TypeA and TypeB**

---

### ✅ Example:

```ts
type Person = {
  name: string;
};

type Employee = {
  empId: number;
};

type PersonEmployee = Person & Employee;

const user: PersonEmployee = {
  name: "Naresh",
  empId: 101
};
```

👉 Here:

* `PersonEmployee` must have **both** `name` and `empId`

# 🟡 2. Why do we use Intersection Types?

### 🎯 Main Reasons:

### 1. Combine multiple responsibilities

```ts
type Logger = {
  log: () => void;
};

type ErrorHandler = {
  handleError: () => void;
};

type System = Logger & ErrorHandler;
```

👉 One object → multiple capabilities

---

### 2. Reusability

Instead of rewriting:

```ts
type User = {
  name: string;
  email: string;
};

type Admin = User & {
  role: string;
};
```

👉 Code clean + reusable

---

### 3. Strong type safety

👉 TypeScript ensures all properties exist

# 🟠 3. Intermediate Level

## ✅ Intersection with Interfaces

```ts
interface A {
  a: string;
}

interface B {
  b: number;
}

type C = A & B;
```

---

## ✅ Intersection with Functions

```ts
type A = {
  a: string;
};

type B = {
  b: number;
};

function printData(data: A & B) {
  console.log(data.a, data.b);
}
```

## ⚠️ Conflicting Types (Important)

```ts
type A = {
  id: string;
};

type B = {
  id: number;
};

type C = A & B;
```

👉 Result:

```
id: never ❌
```

👉 Because:

* string & number = impossible

# 🔵 4. Advanced / Expert Level

## ✅ 1. Intersection with Union (Powerful concept)

```ts
type A = { name: string };
type B = { age: number };

type C = A & (B | { address: string });
```

👉 Meaning:

* Must have `name`
* And either:

  * age OR address

---

## ✅ 2. Mixin Pattern (Real-world usage)

```ts
type Timestamp = {
  createdAt: Date;
};

type SoftDelete = {
  deleted: boolean;
};

type Model = Timestamp & SoftDelete;

const obj: Model = {
  createdAt: new Date(),
  deleted: false
};
```

👉 Common in:

* Backend models
* Database schemas

---

## ✅ 3. Extending Complex Types

```ts
type Base = {
  id: number;
};

type WithAudit = Base & {
  createdBy: string;
  updatedBy: string;
};
```

---

## ✅ 4. Intersection for API Responses

```ts
type Success = {
  success: true;
  data: string;
};

type Error = {
  success: false;
  message: string;
};

type Response = Success | Error;
```

👉 Combine with intersection:

```ts
type WithMeta = {
  requestId: string;
};

type FinalResponse = (Success | Error) & WithMeta;
```

---

# 🔥 Key Difference (Important Interview Point)

| Concept            | Meaning       |                     |
| ------------------ | ------------- | ------------------- |
| Union (`           | `)            | Either this OR that |
| Intersection (`&`) | BOTH required |                     |

---

### Example:

```ts
type A = { a: string };
type B = { b: number };

let x: A | B; // only one needed
let y: A & B; // both needed
```

* Ensures all properties exist

* Used in real-world apps heavily
