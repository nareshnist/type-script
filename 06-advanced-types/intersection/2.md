# 🔵 1. API Response + Metadata (Very Common)

👉 Real-world: Every API response has:

* data
* status
* request info

```ts
type ApiData = {
  users: string[];
};

type Meta = {
  requestId: string;
  timestamp: number;
};

type ApiResponse = ApiData & Meta;

const response: ApiResponse = {
  users: ["Naresh", "John"],
  requestId: "abc123",
  timestamp: Date.now()
};
```

### 🧠 Why?

👉 Backend + frontend both need shared structure
👉 Clean separation + reuse

# 🟢 2. React Props Composition (🔥 Very Important)

👉 Real-world in React apps:

```ts
type ButtonProps = {
  label: string;
};

type StyleProps = {
  color: string;
  size: string;
};

type Props = ButtonProps & StyleProps;

function Button(props: Props) {
  return <button>{props.label}</button>;
}
```

### 🧠 Why?

👉 Split props into:

* logic props
* UI props
  👉 Combine using `&`

# 🟡 3. User Roles System

👉 Real-world: User + Admin

```ts
type User = {
  name: string;
  email: string;
};

type Admin = {
  role: "admin";
  permissions: string[];
};

type AdminUser = User & Admin;
```

### 🧠 Why?

👉 Instead of duplicating user fields
👉 Extend functionality cleanly

# 🔴 5. Database Models (Backend / Full-stack)

```ts
type BaseModel = {
  id: string;
};

type Timestamps = {
  createdAt: Date;
  updatedAt: Date;
};

type User = BaseModel & Timestamps & {
  name: string;
};
```

### 🧠 Why?

👉 Common fields (id, timestamps) reuse
👉 DRY principle follow

# 🟠 6. Feature-based Modular Code

👉 Real-world: Logging + Error handling

```ts
type Logger = {
  log: (msg: string) => void;
};

type ErrorHandler = {
  handleError: (err: Error) => void;
};

type System = Logger & ErrorHandler;
```

---

### 🧠 Why?

👉 Different modules combine → one system
👉 Clean architecture

# 🟢 8. UI State Management (React / Vue)

```ts
type Loading = {
  isLoading: boolean;
};

type Data = {
  data: string[];
};

type State = Loading & Data;
```

---

👉 OR more advanced:

```ts
type ErrorState = {
  error: string | null;

type AppState = Loading & Data & ErrorState;
```

# 🔥 10. Conditional Feature Objects

```ts
type Base = {
  name: string;
};

type WithAge = {
  age: number;
};

type WithAddress = {
  address: string;
};

type Person = Base & (WithAge | WithAddress);
```

---

👉 Meaning:

* name compulsory
* plus either age OR address
